Notification Service

The Notification Service is responsible for handling all communication needs across various applications and services that require sending notifications via SMS, email, and mobile push notifications. It ensures reliable delivery of messages by leveraging a queue-based architecture for scalability and fault tolerance.

Key Responsibilities

  • Receiving notification requests from multiple services and applications
  • Queuing requests in Redis Streams for asynchronous processing
  • Processing queued requests and sending notifications via the appropriate channel
  • Ensuring messages contain the correct contact details and content format

Notification Workflow

The Notification Service follows a queue-based architecture to handle message processing efficiently. Source applications send notification requests to the Receiver API, which appends them to the Redis Message Queue. The Message Processor retrieves these messages and sends them to the appropriate external services based on the notification type.

Step Component Description
1 Source Applications and Services Various applications generate notification requests and send them to the Notification Service.
2 Receiver API The API receives notification requests and appends them to the Redis Message Queue.
3 Redis Message Queue Temporarily holds notification requests, ensuring they are processed sequentially and efficiently.
4 Message Processor Pulls messages from the queue and determines the notification type (Email, SMS, Mobile Notification).
5a Email Handler Sends emails via configured SMTP servers.
5b SMS Handler Forwards SMS notifications to SMS APIs for delivery.
5c Mobile Notification Handler Sends push notifications to users via Google Messaging or similar services.

Request Structure

Contact Details

Notification Type Required Contact Information
SMS Mobile number
Email Email address
Mobile Notification Unique device ID

Message Content

Notification Type Required Content
SMS Text message content
Email HTML content including subject and body
Mobile Notification Header image and notification content

Technology Stack

  • Node.js for handling API logic and message processing
  • TypeScript for type safety and structured development
  • Redis for queueing notification requests and ensuring message durability
  • MongoDB for notification information persistance and reporting

Storage API Service

Introduction

The Storage Service provides a scalable and secure solution for storing, retrieving, and managing files. It allows applications and users to upload, retrieve, and delete files while enforcing access controls based on predefined permissions. The service ensures fine-grained control over data access, integrating with authentication and authorization mechanisms to provide a robust security model.

Advantages of the Storage Service

Advantage Description
Fine-Grained Access Control Permissions are managed at the folder level using Keycloak scopes and MongoDB-based user access rules.
Scalable & Secure Designed to handle large file storage with efficient retrieval and access management.
API-Based Access Provides a structured REST API for seamless integration with applications.
Authentication & Authorization Uses Keycloak to enforce access control with role-based and scope-based permissions.
Multi-Tenant Support Enables isolated storage for different users or services with independent access controls.

Access Control Mechanism

  • Keycloak Scopes:
  • storage.put: Required for uploading files
  • storage.get: Required for downloading files
  • storage.delete: Required for deleting files
  • MongoDB-Based Permissions:
  • Folder-level access rules define which users can perform operations within specific directories.
  • Each file is associated with metadata that determines its ownership and access level.

Technology Stack

Component Description
Node.js API backend for handling file operations and access control
TypeScript Provides type safety and structured development
MongoDB Stores metadata and user permissions for folder-based access control
Object Storage Backend Designed to store and retrieve large files efficiently
Keycloak Handles authentication and authorization using predefined scopes

The Storage Service is designed to provide a secure, scalable, and API-driven approach to file management while ensuring controlled access through Keycloak and MongoDB-based authorization.

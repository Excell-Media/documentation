Infrastructure Overview

Components

The infrastructure of the Excell ecosystem consists of three primary layers:

  • Databases: Responsible for storing structured, semi-structured, and real-time data across various services.
  • Products: Core third-party solutions integrated into the system for authentication, logging, monitoring, and API management.
  • Microservices: Independent services handling different business functions, ensuring modularity and maintainability.

The following table categorizes these components, providing a structured view of the ecosystem.

Category Components
Databases PostgreSQL, MongoDB, Redis, Neo4j
Products Keycloak, Fluent Bit, Prometheus, Grafana, Nginx
Microservices Core API Service, Keycloak Admin API, Notification API, Storage API, External API Service, Observability Service

Deployment Approach

The Excell infrastructure can be deployed using two different approaches: Kubernetes-based deployment and Standalone VM deployment. Each method offers distinct advantages and is suited for different operational and scaling needs.

  • Kubernetes Deployment: In this model, microservices and supporting components run as containerized workloads orchestrated across multiple VMs. Kubernetes ensures automated scaling, fault tolerance, and resource management, providing a dynamic and resilient infrastructure.
  • Standalone Deployment: This approach involves running services directly on virtual machines (VMs) without containerization. Services are strategically distributed across VMs to optimize resource utilization while maintaining manual control over deployments, networking, and scaling.

The choice between these two deployment strategies depends on factors such as scalability, operational complexity, and resource availability. In the following sections, we will explore how each deployment model is structured, how the components are distributed, and the trade-offs involved. The final section will compare both methods, highlighting their pros and cons to assist in selecting the most suitable deployment strategy.

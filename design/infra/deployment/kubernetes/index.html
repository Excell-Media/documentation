Kubernetes Deployment for Excell

The Excell ecosystem is deployed on a Kubernetes cluster, ensuring scalability, resilience, and efficient resource management. This setup is designed to handle 10,000 employees (3,000 concurrent logins) and 200,000 broadband customers while adhering to best practices for container orchestration.

Note: The Keycloak and PostgreSQL setup is handled separately and is not included in this deployment plan.

Kubernetes Deployment Overview

The Excell application stack consists of the following components:

  • Databases: MongoDB, Redis, Neo4j
  • Products: Fluent Bit, Prometheus, Grafana, Nginx
  • Microservices: Core API, Keycloak Admin API, Notification API, Storage API, External API, Observability Service

To ensure optimal performance and fault tolerance, these components are distributed across a bare minimum of 11 virtual machines (VMs) within the Kubernetes cluster.

Infrastructure Setup

1. Control Plane Nodes

Component Description
Kubernetes API Server Central management interface for all cluster operations.
Controller Manager Ensures the cluster's desired state is maintained.
Scheduler Assigns workloads (pods) to worker nodes based on resources.
etcd Distributed key-value store for Kubernetes configuration data.
  • Number of VMs: 3 (Ensures high availability and fault tolerance)
  • CPU & RAM Estimate: 4 vCPU, 16GB RAM each

2. Database Nodes

Database Description
MongoDB Handles document-based storage for various microservices.
Redis Acts as a caching layer to improve response times.
Neo4j Graph database for managing network infrastructure inventory.
  • Number of VMs: 3
  • CPU & RAM Estimate: 8 vCPU, 32GB RAM each

3. Microservices Nodes

Microservices Description
Core API Manages business logic and application workflows.
Keycloak Admin API Handles IAM-related admin operations.
Notification API Processes and sends notifications via SMS, Email, and Mobile Push.
Storage API Provides file storage operations with access control.
External API Interfaces with third-party services such as Aadhaar authentication.
Observability Service Manages logging, metrics, and system health monitoring.
  • Number of VMs: 3
  • CPU & RAM Estimate: 8 vCPU, 32GB RAM each

4. Observability Nodes

Observability Components Description
Fluent Bit Collects and forwards logs from all services.
Prometheus Monitors and scrapes system and application metrics.
Grafana Provides dashboards for real-time system insights.
  • Number of VMs: 2
  • CPU & RAM Estimate: 4 vCPU, 16GB RAM each

5. Load Balancer Node

Component Description
Nginx Ingress Controller Manages external traffic routing for Kubernetes services.
  • Number of VMs: 1
  • CPU & RAM Estimate: 4 vCPU, 16GB RAM

Storage and Persistence

  • MongoDB requires persistent volumes (PV) and persistent volume claims (PVC) for data integrity.

Minimum Kubernetes VM Allocation

VM Group No. of VMs CPU Allocation RAM Allocation
Control Plane Nodes 3 4 vCPU each 16GB RAM each
Database Nodes 3 8 vCPU each 32GB RAM each
Microservices Nodes 3 8 vCPU each 32GB RAM each
Observability Nodes 2 4 vCPU each 16GB RAM each
Load Balancer Node 1 4 vCPU 16GB RAM
Total 11 VMs ~76 vCPU ~352GB RAM

This Kubernetes deployment model ensures reliability, scalability, and security while optimizing resource utilization for Excell’s high-performance microservices ecosystem.

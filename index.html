Excell Architecture Documentation

Welcome to the Excell Architecture Documentation, detailing the infrastructure, application architecture, and deployment strategies used to build a scalable, efficient, and reliable system. This documentation provides a structured overview of how various components integrate to support 10,000 employees (3,000 concurrent logins) and 200,000 broadband customers efficiently.

Infrastructure

The infrastructure is designed to provide resilience, scalability, and cost optimization, supporting various deployment models.

  • Infrastructure Overview – Summarizes all components in Excell’s infrastructure.
  • Deployment Strategies
    • Deployment Overview – Covers the two deployment strategies available.
    • Kubernetes Deployment – Explains how Excell is deployed in a containerized environment.
    • Standalone Deployment – Details a VM-based approach for running services efficiently.

Application Architecture

The application layer consists of various microservices and supporting components that enable business processes.

  • Application Overview – Provides a high-level view of the Excell application structure.
  • Identity & Access Management (IAM)
    • IAM Implementation – Details how user authentication and authorization are managed with Keycloak.
    • User Administration API – API for managing users and roles.
  • Inventory Management
    • Graph-Based Inventory System – Explains how Neo4j is used for network infrastructure management.
  • Microservices Architecture
    • Microservices Overview – Covers the microservices ecosystem and their interactions.
    • Core Microservices
      • Core API Service – Handles core business logic.
      • Notification API – Manages SMS, email, and mobile notifications.
      • Storage API – Provides file management and access control.
      • External API Service – Manages third-party integrations.
      • Observability Service – Monitors logs, metrics, and system performance.
